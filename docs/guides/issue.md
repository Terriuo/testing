**üêõ How to Submit a Bug Report**

Use this template to report any bugs.

**1\. Title:** Your title should be descriptive and follow this format: \[BUG\] - (Section of the App) (Your Platform) _Example: \[BUG\] - User Profile (iOS)_

**2\. Fill out the form:**

- **Environment**: Select if you found the bug in Production or Development.
- **Platform**: Choose all platforms where you experienced the bug (e.g., iOS (Device), Android (Simulator)).
- **Version**: Specify the exact version of the application you are using (e.g., 1.2.1).
- **Bug Description & Steps to Reproduce**: Please be as detailed as possible.
  - **Steps to Reproduce**: Provide a clear, step-by-step list of actions to trigger the bug.
  - **Expected Behavior**: Describe what you thought should happen.
  - **Actual Behavior**: Describe what actually happened.
  - **Screenshots or videos**: If you can, please attach images or a screen recording to help illustrate the problem.
- **Logs**: If you have access to any error logs, please paste them here.

**‚ú® How to Submit a Feature Request**

Use this template to suggest a new feature or an improvement to an existing one.

**1\. Title:** Start your title with \[FEATURE\] - followed by a short description of your idea. _Example: \[FEATURE\] - Add dark mode support_

**2\. Fill out the form:**

- **Is your feature request related to a problem?**: Describe the problem you're facing that this feature would solve.
- **Describe the solution you'd like**: Provide a clear description of what you want to happen and how you envision the feature working.
- **Additional Context**: Add any other context that might help explain your feature request.

**‚úÖ How to Create a Task**

Use this template to outline a specific, actionable task that needs to be completed.

**1\. Title:** Start your title with \[TASK\] - followed by a brief summary of the task. _Example: \[TASK\] - Update user authentication documentation_

**2\. Fill out the form:**

- **Task Description**: Provide a detailed description of the task, its goals, and what needs to be done.
- **Acceptance Criteria**: Define a clear checklist of requirements that must be met for the task to be considered complete.
  - Example:
    - \[ \] Code is written and reviewed.
    - \[ \] Unit tests are added and passing.
    - \[ \] Documentation is updated.
